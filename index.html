Context

Beautery is a beauty and health marketplace. In 2025, the team launched a referral program for experts: they curate product collections, share them, and earn commissions - both from their own sales and from the sales of experts they recruit.

The Task: Design a personal dashboard that helps experts: track and manage earnings, share referral links, and manage product collections.

Constraints: Using the existing design system, reusing ready-made front-end and back-end components, and limited resources.

Team: 1 Frontend Developer, 1 Backend Developer, 1 Product Designer (Me)

Success Metrics

For the initial launch, the primary success metric was the quality of the user experience. We evaluated how easily users completed key scenarios and monitored the conversion rate for creating the first collection.

My Contribution

I managed the full-cycle design process - conducted discovery, mapped user journeys, built prototypes and components, performed usability testing, wrote UX copy, and designed illustrations for banners and empty states. I led the project by communicating with stakeholders, the technical team, and users.

Discovery & Research

Conducted stakeholder interviews and competitive analysis to understand the referral ecosystem. Identified key pain points in existing solutions and opportunities for differentiation.

Dashboard Design

Created a comprehensive dashboard interface that provides experts with real-time analytics, commission tracking, and partner management tools. The design emphasizes clarity and efficiency for daily operations.

Collection Creation Flow

Streamlined the process for experts to create and manage product collections, reducing cognitive load and minimizing steps required to complete common tasks.

Financial Management

Developed intuitive financial tracking interfaces that help experts monitor earnings, view commission breakdowns, and manage payment preferences with transparency and clarity.

Profile Management

Designed comprehensive profile systems that allow experts to showcase their expertise, manage their public presence, and build trust with potential brand partners.

Alert System

Implemented contextual alert dialogs and notification systems to guide users through complex processes and provide timely feedback on their actions.

Validation States

Created comprehensive validation and error state designs to ensure users receive clear feedback when input validation fails or system errors occur.

Empty States

Designed thoughtful empty states that guide users toward next actions and maintain engagement even when no data is available.

Mobile-First Design

Ensured optimal mobile experience with responsive design patterns and touch-friendly interfaces that maintain functionality across all device sizes.

Outcome

Launched a platform where experts can earn and recruit new partners. The most significant improvement came from simplifying navigation through consistent naming and adding contextual UI tooltips.

Product Growth

The backlog includes developing public storefronts to boost organic traffic and implementing granular click-level analytics.

Summary & Reflection

I managed the full design cycle under tight deadlines and resource constraints. This project validated the impact of a systems-thinking approach and usability testing. As my first experience as a solo designer on a complex product, it further strengthened my interest in designing B2B ecosystems.

Marketplace Admin Dashboard

Intro

I worked within a product team on an internal back-office for a large-scale marketplace. My primary focus was the Content Management & Global Settings segments. I designed end-to-end scenarios for managing marketplace content, as well as core system configurations, including logo, contact data, and user role management.

30% faster Time-to-Market

Achieved by leveraging an open-source design system

Reusable System Solution

Later scaled across 2 other projects: Costo and Beautery

Challenge

The team operated under three critical constraints:

  • Delivering functional modules in short time
  • Ensuring the architecture could support future feature expansion
  • Streamlining internal workflows for content managers

Cross-Functional Collaboration

The project was a collaboration between four designers, each managing a specific functional segment. We maintained a tight feedback loop with engineering, regularly synchronizing design decisions to validate technical feasibility and aligning UI components with existing system capabilities and API architecture.

Design & Logic

To accelerate development and ensure consistency, we used Ant Design as our foundational framework, extending it to fit specific product scenarios. Our design logic for all data management followed a unified CRUD-based structure, allowing users to:

  1. Create Items — through structured forms
  2. Read Items — via data-driven tables
  3. Update Items — using intuitive modals
  4. Delete Items — with confirmation dialogs

Data-Driven Tables

Implemented comprehensive data tables with filtering, sorting, and pagination capabilities. Designed intuitive form logic for complex data entry scenarios.

Intuitive Form Logic

Designed streamlined form workflows that reduce cognitive load and minimize user errors. Implemented smart validation and contextual help systems.

Bulk Operations

Implemented efficient batch processing capabilities for large-scale content operations, significantly reducing operational overhead.

Unified Interface States

Consistent states (Loading, Empty, Error, Confirmation) were applied across all components to ensure a predictable user experience.

Results

We successfully launched a scalable administration system with robust content management modules. By prioritizing reusable components and data-driven logic, we accelerated the development cycle and ensured long-term maintainability. This framework proved so effective that it was subsequently reused for two other e-commerce launches within the company.
